# Артемида-2: Симуляция траектории полёта

Проект для моделирования траектории свободного возврата космического корабля «Орион» в рамках миссии Artemis II. Проект состоит из физического движка на C++ и визуализации телеметрии на Python.

# TL;DR
Вот основные компоненты системы:
* Физический движок (C++): Численное интегрирование (метод Рунге-Кутты 4) и расчет гравитационных сил Земли и Луны - реализовано в **src/physics.cpp**.  
* Парсинг и логика симуляции (C++): Парсер телеметрии NASA Horizons, алгоритм поиска пика скорости для определения точки выключения двигателей (TLI) и главный цикл симуляции - находятся в **src/main.cpp**.  
* Визуализация (Python): Скрипт двойной анимации траекторий на базе Pandas и Matplotlib — лежит в **scripts/plot_orbit.py**.  
* Сборка: Весь жизненный цикл проекта (CMake конфигурация, компиляция, запуск расчетов и визуализации) полностью автоматизирован скриптом **run.py**.

## Архитектура и стек технологий

* **C++:** Отвечает за парсинг эфемерид и математическое моделирование полета методом численного интегрирования.
* **Python:** Скрипты на базе `Pandas` и `Matplotlib` для отрисовки двойной анимации.
* **CMake:** Кроссплатформенная сборка проекта (Windows / Linux).

## Математическая модель

В основе симуляции лежит решение задачи трех тел (Земля, Луна, Орион) с использованием **Метода Рунге-Кутты 4-го порядка**. 

**Сравнение симуляции с реальными данными NASA:**
Уникальной особенностью проекта является параллельное сравнение траекторий. Расчетная траектория выводится на экран синхронно с реальной телеметрией NASA, что позволяет визуально оценить расхождения идеализированной физической модели по сравнению с реальным N-body моделированием. 

Погрешность в расчетах возникает в силу того, что данная модель не берет во внимание гравитационное влияние других тел Солнечной системы, давление солнечного света, а также микрокорректировки траектории посредством включения двигателей Ориона на маршевом участке.

## Как запустить проект

Вы можете либо воспользоваться готовым скомпилированным релизом (рекомендуется для быстрого ознакомления), либо собрать проект вручную из исходного кода.

### Вариант 1: Быстрый запуск 

Для этого способа не требуются CMake и компилятор C++. Достаточно скачать архив под вашу ОС со страницы **[Releases](https://github.com/GolovinD1/artemis2-simulation/releases/tag/v1.0.0)**:

**Windows:**
1. Скачайте архив `Artemis2_Windows.zip` и распакуйте его.
2. Запустите скрипт `run_sim.bat` двойным кликом или через командную строку.

**Linux:**
1. Скачайте архив `Artemis2_Linux.tar.gz` и распакуйте его.
2. Откройте терминал в папке с проектом и запустите скрипт:
       ```bash
       chmod +x run_sim.sh  # Если скрипту не хватает прав на исполнение
       ./run_sim.sh
       ```

> **Примечание:** Для работы визуализации на компьютере должен быть установлен только Python 3 с необходимыми библиотеками (`pandas`, `numpy`, `matplotlib`).

### Вариант 2: Сборка из исходников

**Требования:**
* Установленный **компилятор C++** (Visual Studio 2022, GCC или Clang)
* **CMake** (версия 3.15 и выше)
* **Python 3** с библиотеками `pandas`, `numpy`, `matplotlib`

Проект использует систему сборки CMake. Все временные файлы компиляции генерируются в папку `build/`, которая создается автоматически.

Для запуска полного цикла (конфигурация -> сборка -> расчет физики -> показ графиков) просто откройте терминал в корне проекта и выполните команду:
```bash
python run.py
```
Этот скрипт автоматизирует процесс конфигурации, сборки и запуска обоих модулей.

Или можете использовать другой метод - запуск через терминал поочерёдно:

```bash
cmake -B build
cmake --build build

# Запуск C++ кода (на Linux/macOS)
./build/artemis_sim

# Запуск C++ кода (на Windows)
.\build\Debug\artemis_sim.exe

# Запуск графиков - Python
python scripts/plot_orbit.py
```

## Источники данных
Реальные эфемериды для Луны и корабля «Орион» получены из официальной базы данных **NASA JPL Horizons**.
